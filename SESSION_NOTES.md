## Challenges & Issues Encountered

### 1. Spotify 404 "No Active Device" Errors
**Problem:** User could control playback (skip, pause) but couldn't start new playback via chat.

**Root causes discovered:**
- Passing empty string `""` for device_id sometimes fails
- Tried auto-detecting devices with `getDevices()` but this also had issues
- **Final solution:** Simplified to match working controls - pass empty string and let Spotify pick device

**Current state:** Should work, but may still be flaky. The controls use the same approach and work.

### 2. Rate Limiting (429 Errors)
**Problem:** Polling was too aggressive (every 3 seconds), hitting Spotify rate limits.

**Solution:** Slowed polling in `lib/hooks/use-playback-polling.ts`:
- Active: 3s → 5s
- Paused: 10s → 15s
- Max backoff: 30s → 60s

### 3. Recommendations API 404
**Problem:** `seed_artists=&seed_tracks=` being passed as empty strings caused 404.

**Solution:** In `lib/spotify/recommendations.ts`, only include non-empty seed arrays:
```typescript
if (seedArtists.length > 0) requestParams.seed_artists = seedArtists;
if (seedTracks.length > 0) requestParams.seed_tracks = seedTracks;
if (seedGenres.length > 0) requestParams.seed_genres = seedGenres;
```

### 4. Auth Callback Infinite Spinner
**Problem:** After OAuth callback, `isLoading` stayed `true` forever.

**Solution:** In `components/auth/auth-provider.tsx`, added `await initialize()` after `exchangeCodeForTokens()`.

### 5. Environment Variables with Comments
**Problem:** `.env.local` had comments on same line as values:
```
ELEVENLABS_VOICE_ID=xyz    # comment
```
This included the comment in the value!

**Solution:** Comments must be on separate lines in `.env` files.

## Current Known Issues

### 1. Voice DJ Plays Through Browser, Not Sonos
The DJ voice uses browser `<audio>` element. To play through Sonos would require:
- Sonos Cloud API integration, OR
- System audio routing to Sonos, OR
- A different architecture (serve audio URL that Sonos can play)

### 2. Spotify Device Detection Still Fragile
If user has multiple devices or Spotify isn't actively playing, playback commands may fail. The app assumes an active device exists.

### 3. Environment Variables Must Be Set in Vercel
`.env.local` is gitignored. For production:
- All env vars must be set in Vercel dashboard
- Settings → Environment Variables
- Then redeploy

**Required env vars for production:**
```
ANTHROPIC_API_KEY=sk-ant-...
SPOTIFY_CLIENT_ID=...
SPOTIFY_CLIENT_SECRET=...
SPOTIFY_REDIRECT_URI=https://tunes.dylandibona.com/api/auth/callback/spotify
ELEVENLABS_API_KEY=sk_...
ELEVENLABS_VOICE_ID=rWyjfFeMZ6PxkHqD3wGC
NEXT_PUBLIC_SPOTIFY_CLIENT_ID=...
NEXT_PUBLIC_APP_URL=https://tunes.dylandibona.com
```
